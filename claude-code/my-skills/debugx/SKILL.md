# debugx — 分析日志内容，找出错误原因并尝试进行修复

用户调用了 `/debugx` 并提供了以下参数：

- $0 ：运行出现问题的命令，默认在项目根目录下执行。
- $1 ：可选的输出日志文件路径，如果用户提供了这个参数，优先使用它来分析日志内容，否则使用默认的输出日志文件路径 logs/debug_output.log。

# 执行步骤

1. 根据运行命令判断当前在进行训练、推理、评测，还是简单的测试分析。

2. 根据日志文件路径读取日志内容，分析日志中的错误信息，找出错误原因。

3. 根据错误原因尝试进行修复，提供具体的修复建议或者修改命令。

4. 在修复后根据用户需要重新运行命令或结束任务。
